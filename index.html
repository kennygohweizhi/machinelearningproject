Background

Using devices such as Jawbone Up, Nike FuelBand, and Fitbit it is now possible to collect a large amount of data about personal activity relatively inexpensively. These type of devices are part of the quantified self movement â€“ a group of enthusiasts who take measurements about themselves regularly to improve their health, to find patterns in their behavior, or because they are tech geeks. One thing that people regularly do is quantify how much of a particular activity they do, but they rarely quantify how well they do it. In this project, your goal will be to use data from accelerometers on the belt, forearm, arm, and dumbell of 6 participants. They were asked to perform barbell lifts correctly and incorrectly in 5 different ways. More information is available from the website here: http://groupware.les.inf.puc-rio.br/har (see the section on the Weight Lifting Exercise Dataset).

We first read the two datasets:

submission = read.csv("pml-testing.csv")
train = read.csv("pml-training.csv")

Partitioning the training set

We separate the training set into a training and testing set, to validate the model

set.seed(3333)
sampling = createDataPartition(train$classe, p = 0.8, list=F)
training = train[sampling,]
testing = train[-sampling,]
dim(training); dim(testing)

We have 15,699 observations in the training set, and 3,923 in the testing set. There are 160 variables.


Creating covariates: NA terms

First, we remove variables where NAs comprise over half of its observations

rmNA = which(apply(training, 2, function(x) sum(is.na(x)))> nrow(training)/2)
procTraining = training[,-rmNA]
procTesting = testing[,-rmNA]
procSubmission = submission[,-rmNA]

Creating covariates: near zero variance terms

Second, we exclude variables which have near zero variance

nearZeroCol = nearZeroVar(procTraining)
procTraining = procTraining[,-nearZeroCol]
procTesting = procTesting[,-nearZeroCol]
procSubmission = procSubmission[,-nearZeroCol]

Creating covariates: descriptive variables

Lastly, we drop the first six variables which are descriptive variables 

procTraining = procTraining[, -c(1:6)]
procTesting = procTesting[, -c(1:6)]
procSubmission = procSubmission[, -c(1:6)]

Dimension reduction using PCA 

dim(procTraining)

We still have 53 variables, which will take some time to process. Hence, we reduce the number of covariates using PCA. 

pca = preProcess(procTraining[,-53], method="pca", thresh=0.9)
pca

We now have 18 predictors


Model fitting

We will use random forest.

modelTREE = train(procTraining$classe~., data=pcatraining, method="rf")

Out of sample accuracy

Test the model on the testing dataset.

confusionMatrix(predict(modelTREE, pcatesting), procTesting$classe)

The results are shown below. We have achieved an accuracy rate of 97.99%.

pml_write_files = function(x) {
    n = length(x)
    for (i in 1:n) {
        filename = paste0("problem_id_", i, ".txt")
        write.table(x[i], file = filename, quote = FALSE, row.names = FALSE, 
            col.names = FALSE)
    }
}

pml_write_files(output)
